# 鱼群养殖预测分析 —— 需求分析文档

> 编制日期：2026-07-17（v3，对齐原型最新版本）
> 平台：智慧海洋养殖管控平台（福建海域网箱养殖，6 个站点）
> 范围：盘点已接入/待接入数据，规划鱼群养殖预测分析能力，输出养殖建议
> 说明：水下光学摄像头因浊水场景参考意义有限，**本期起弃用**；鱼群计数/测长改由声纳承担。
> 原型：HTML 原型已完成（`prototype/` 目录，12 个页面 + 导航页），覆盖数据看板、知识库、智能体配置、AI 智能分析/问答/副驾等，见第九章。
>
> **数据层级模型**：`养殖站点 → 养殖单元（鱼排 4m×4m / 格，16㎡）→ 鱼群 + 投喂`。每个站点由若干养殖单元组成（当前 5 个生产站点合计 63 个单元、放养约 30,290 尾），**放养数量、声纳计数/测长、投喂量均按养殖单元独立记录与统计**，站点级数据为单元汇总。
> - 声纳按单元获取鱼情：核心单元用固定声纳连续监测，其余用 1 台声纳轮巡扫描相邻单元；存活率 = 声纳计数 / 放养数量（当前约 98.6%）。

## 四、鱼群养殖预测分析方案（分两期落地）

> **落地策略（关键）**
> - **一期（当前）**：数据尚未充分积累，**先引入通用文本大模型（LLM）+ 知识库 + 规则做分析与建议**。把各站点实时数据（水质/气象/投喂/声纳计数等）结构化拼接成上下文喂给大模型，由大模型输出**自然语言的投喂建议、养殖建议、风险评估**；同时建设**指标报表/数据看板**。优势：无需训练、上线快、可解释。
> - **二期（数据充足后）**：在历史数据沉淀到位后，再训练**专用小模型**（生长曲线、生物量、低氧预测、病害风险等），做更精准的量化预测。小模型的预测结果仍可回流给大模型做解读和报告生成。

### 4.1 预测分析目标场景（7 项）

| # | 预测场景 | 业务价值 | 期次 |
|---|---------|---------|------|
| 1 | **存塘生物量/产量预测** | 估产、销售排期、资金安排 | 🟢一期LLM粗估 → 🔵二期小模型精算 |
| 2 | **生长曲线与达规格时间预测** | 预计何时达到上市规格、上市时间 | 🔵二期（需体长时序） |
| 3 | **投喂优化与 FCR 预测** | 降低饲料成本、减少残饵污染 | 🟢一期LLM建议（规则+经验） |
| 4 | **溶解氧低氧风险预测** | 预防泛塘死鱼，提前增氧/减投 | 🟢一期规则预警 → 🔵二期DO时序模型 |
| 5 | **病害/死亡风险预测** | 早期预警、降低损失 | 🟢一期LLM风险评估 → 🔵二期 |
| 6 | **网衣破损/逃鱼风险预测** | 防逃鱼、维保排期 | 🟢一期声纳巡检+LLM → 🔵二期 |
| 7 | **极端天气损失预测** | 台风/高温前决策（加固、提前起捕） | 🟢一期LLM应对建议 |

### 4.2 预测分析的"五步法"（一期 LLM 卡片结构）

每条预测建议按以下五步呈现，保证逻辑可追溯：

```
数据 → 经验 → 分析 → 预测 → 建议
```

- **数据**：列出融合了哪些数据源（水质/气象/声纳/投喂…）
- **经验**：引用知识库规则/经验（如 DO<3 泛塘、体长体重公式）
- **分析**：综合推理（数据 + 经验说明什么）
- **预测**：预测会发生什么（带量化）
- **建议**：可执行处置

### 4.3 各预测项的方法 / 模型

#### ① 存塘生物量预测
- 输入：声纳**鱼群计数 N** + **体长分布** → 体长体重公式 `W = a·Lᵇ`（按品种取文献参数）→ 单尾均重 → **生物量 = N × 均重**
- 扣减：累计死亡（死鱼记录/声纳沉鱼）、累计起捕
- 输出：当前存塘量（kg/尾）、产量预测曲线

#### ② 生长曲线与达规格时间预测
- 模型：**von Bertalanffy 生长方程** `L_t = L_∞(1−e^(−K(t−t₀)))` 或**热单位/积温模型**（水温驱动生长）
- 输入：历史体长时序（声纳测长）+ 水温积温 + 投喂量
- 输出：未来体长曲线、达到上市规格（如大黄鱼 ≥ 某克重）的预计日期

#### ③ 投喂优化与 FCR 预测
- 输入：水温、溶解氧、鱼群体长/生物量、**投喂时声纳摄食响应**（吃食活跃度）
- 规则 + 模型：水温/DO 决定基础投喂率表，声纳摄食响应动态修正（吃不完则减量）
- 输出：每日推荐投喂量、预测 FCR、残饵风险提示

#### ④ 溶解氧低氧风险预测
- 原理：水温↑→饱和溶氧↓；夜间呼吸耗氧；大潮/天气影响
- 模型：历史 DO 时序（LSTM / Prophet）+ 水温 + 天气预报 + 潮汐 → 未来 6–24h DO 曲线
- 输出：低氧风险等级、增氧/减投建议时刻

#### ⑤ 病害/死亡风险预测
- 输入：水质异常（DO 低、pH 异常、水温骤变）、气象应激、死鱼上升趋势、鱼群行为异常（声纳）
- 方法：阈值规则 + 风险评分模型（历史告警/死亡记录训练）
- 输出：风险等级与排查建议

#### ⑥ 网衣破损/逃鱼风险预测
- 输入：声纳网衣巡检结果 + 流速/浪高 + 网衣使用年限
- 输出：破损概率、巡检/换网排期

#### ⑦ 极端天气损失预测
- 输入：台风路径/强度预报、高潮位、大浪、水温
- 输出：风险等级、提前起捕/加固决策建议

## 五、养殖建议输出（平台智能化）

> **一期由通用文本大模型生成**（见第八章），二期起逐步由小模型量化预测 + 大模型解读。建议形态为**可执行的自然语言建议卡**，推送给养殖员 App/后台。

### 5.1 日常投喂建议
- 根据水温、DO、鱼群生物量输出**当日推荐投喂量与投喂时刻**
- 声纳摄食响应不佳时提示**减量或停喂**，降低残饵与 FCR
- 高温/低氧时段避开投喂

### 5.2 水质与增氧建议
- 预测低氧时段提前**开启增氧设备/微孔增氧**
- 盐度异常波动提示**检查淡水注入或换水**
- pH 偏离提示传感器校准或水质干预

### 5.3 生长与上市建议
- 体长分布显示分化大时建议**分箱/分级**
- 达规格比例满足时建议**安排起捕上市**，给出预计产量与最佳上市窗口

### 5.4 风险与应急建议
- 台风/大浪预警：**加固网衣、降低水位、必要时提前起捕**
- 大潮期：加强网衣巡检与溶氧监控
- 死亡率上升/行为异常：提示**停料、巡检、必要时用药**并上报

### 5.5 设备与数据维护建议
- 传感器离线/数据异常（如连江渔排 pH/DO 异常）提示**现场校准/更换**
- 声纳定期**网衣扫描计划**自动排程

## 六、分期实施建议

### 一期：大模型分析 + 指标报表（快速出价值，不训练模型）

**目标**：在数据尚未充分沉淀时，借助通用文本大模型 + 知识库 + 数据看板，先让平台"能看、能分析、能给建议"。

| 工作项 | 内容 |
|--------|------|
| 数据补全 | 接入机器投喂、放苗/起捕/死亡等人工录入；修复水质异常传感器（如连江渔排）；声纳试点 1–2 个站点出计数/测长 |
| **指标报表/数据看板** | 见第七章报表清单，基于现有数据直接出图出表 |
| **知识库建设** | 结构化（品种×规格×类别）+ 非结构化资料 RAG，注入大模型 Prompt（见 8.1） |
| **引入通用文本大模型** | 见第八章，结构化拼接实时数据 → LLM 输出投喂/养殖建议与风险评估 |
| 交付物 | 多维度数据看板 + 每站点 AI 养殖建议卡 + 风险预警提示 + 报表中心 |

### 二期：专用小模型（数据充足后精量化）

**目标**：历史时序数据沉淀到位（体长时序、水质时序、投喂-增重对应关系等）后，训练专用小模型替换/增强一期的经验判断。

| 工作项 | 内容 |
|--------|------|
| 生长曲线模型 | von Bertalanffy / 积温模型，输出达规格上市时间 |
| 生物量精算模型 | 计数×体长体重，结合死亡/起捕扣减 |
| 低氧时序预测 | LSTM/Prophet，提前 6–24h 预测 DO |
| 病害/死亡风险模型 | 风险评分，历史告警/死亡记录训练 |
| 网衣/逃鱼风险模型 | 声纳巡检 + 浪流受力建模 |
| 大模型与小模型协同 | 小模型出量化预测 → 大模型解读为自然语言建议与报告 |

> 一期到二期的切换判据：① 体长/生物量时序累计 ≥ 1 个养殖周期；② 水质时序连续 ≥ 3–6 个月且数据质量达标；③ 投喂-增重数据可对齐。满足后逐项替换为小模型。

## 八、一期 AI 智能化方案（已原型化）

> 一期不训练模型，以**通用文本大模型 + 知识库 + 规则**为核心，已落地以下模块（原型见 `prototype/`）。

### 8.1 知识库管理（结构化 + 非结构化）
- **结构化知识**：按 **品种（大黄鱼/鲈鱼/石斑鱼/真鲷）× 规格（体长 cm 区间）× 类别（水质阈值/投喂经验/告警规则/品种生物学/生长参数/病害应急）** 三维归类；每条可单独启停、按用途（大模型 Prompt / 规则引擎）分发。
  - 水质阈值表、投喂经验表按品种+规格动态渲染；告警规则/生长参数/病害应急等分类有详细列表。
  - 支持手动录入 + **导入结构化文件**（Excel/CSV/JSON，表头匹配字段自动解析）。
- **非结构化资料（RAG）**：导入 PDF/Word/Excel/TXT/Markdown/图片(OCR) → 解析→切片→向量化（bge-large-zh 等，1024 维）→ 混合检索（向量+关键字）；与结构化知识一起参与大模型检索。
- **一键注入大模型**：把知识库（结构化 + 非结构化 chunks）注入 system prompt，全站智能体生效。详见《一期知识库-大黄鱼养殖.md》。

### 8.2 分析智能体配置（可视化）
每个智能体可配置（已内置 4 个示例：每日养殖建议 / 低氧实时预警 / 周报月报 / 台风应急）：
- 名称、**分析类型**（投喂+风险/风险评估/报表分析/应急/**自定义**）、大模型（GLM/通义/DeepSeek 等）
- 关联知识库、分析范围（全部/异常站点）、注入数据（水质/投喂/气象/声纳）
- System Prompt（专家角色，按类型切换模板）
- **触发策略**：定时（频率 每日/每周/每月/自定义cron + 时刻，**可配置**）/ 异常触发（条件）/ 输出形式 / 推送渠道
- **结果展示位置（多选）**：AI 智能分析 / 告警中心 / 报表中心 / AI 智能问答 / 副驾推送 / 弹窗
- 试运行（选站点即时看大模型输出）；列表点击"编辑"切换加载配置

### 8.3 AI 智能分析（定时任务结果展示）
- 浏览"每日养殖建议"智能体定时生成的分析归档：**日期 + 场次**两维度
  - **场次（早场/晚场）来自智能体配置的定时触发时刻**（联动：在智能体配置改时刻，此处场次选项自动同步；早场<12:00 基于夜间/黎明数据，晚场基于白天+明日预报）
- 内容随**顶栏全局站点选择**联动（选站看该站分析，选全部看全站）
- 预测卡按 **数据 → 经验 → 分析 → 预测 → 建议** 五步呈现，按站点分组
- 含全局通用提示 + 各站点专属 24h 提示（各站水温不同，分别给出）

### 8.4 AI 智能问答
- 左侧**多会话页签**（新建/切换/关闭，各自独立记录）
- 咨询范围跟随**顶栏全局范围**（无单独下拉框），输入框/发送按钮固定底部
- 大模型结合实时数据 + 知识库回答（投喂量、溶氧处置、上市时间等）

### 8.5 AI 养殖副驾（全站悬浮）
- 每页右下角悬浮 🤖 入口，点击展开对话面板
- **独立咨询范围选择**（全部/某站点），回复带范围上下文
- **未读角标**与"每日养殖建议"定时任务联动：有新建议亮红点，点开或查看 AI 智能分析页即清除

### 8.6 全局站点选择（全站联动）
- 顶栏「全局范围」下拉框（localStorage 跨页持久化）
- 切换后：水质/投喂/生物量/告警/站点/AI 智能分析等页面**按站点筛选或聚焦**；每页右上角"🔍 已筛选：xxx"标签实时反映
- AI 智能问答、副驾、AI 智能分析的回复/内容均跟随

### 8.7 报表 AI 智能分析（多维度）
报表中心的 AI 智能分析按 **5 个维度** 输出（日/周/月/周期报各自定制）：
- 📊 总体表现（生物量/FCR/存活率等总览）
- 💧 水质与风险（DO/水温/盐度异常、低氧泛塘风险）
- 🍽️ 投喂与生长（投喂量、摄食、体长进展）
- ⚙️ 设备与数据（传感器/设备健康、数据质量）
- 🔮 趋势/预测（后续走势、上市/产量预判）

### 8.8 上下文拼接与大模型选型
- **大模型选型**：通用文本大模型均可（GLM/通义/文心/DeepSeek），建议支持结构化输出（JSON）便于前端渲染。
- **上下文样例**（连江渔排）：
```
【站点】连江渔排  【时间】2026-07-15 22:45
【水质】水温25.8℃ / pH6.27 / DO3.9mgL(饱和度56.8%) / 盐度30.65ppt
【参考阈值】大黄鱼：pH7.8–8.5、DO>5mgL、水温18–28℃、盐度25–32ppt
【气象】晴28℃ 东南风4级 湿度79%；明日多云34/26℃
【潮汐】下次低潮05:30(1.7m) 高潮10:39(5.74m)
【投喂】今日累计8kg(已暂停)，投喂率1.0%BW
【鱼群】声纳计数 4,860 尾，体长均值 20.8cm
【设备】光照传感器离线；其余在线
```
- **输出建议样例**：⚠️ 连江渔排 DO 3.9/pH 6.27 与邻站背离，疑似传感器故障 → 暂停投喂、立即增氧、校准 pH/DO 传感器、凌晨加密巡检。
- **数据质量门禁**：传感器异常/离线时 Prompt 标注，避免基于错误数据给误导建议（连江案例）。
- **与报表联动**：周报/月报的文字分析由大模型基于报表数据自动生成。
